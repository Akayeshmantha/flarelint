Extensions Section

Specifies the types of files to which to apply the rule. You specify types of files with their file name extensions, without the initial dot (“.”). You must specify at least one type of file. To specify more than one type of file, separate file name extensions with spaces and line breaks. A rule file must contain one extensions section. This section must be after the rule section and before the when sections.

Remember that applies rules any XML and XML-based files in a Flare project. does not scan binary files or other text files. You can see a file’s extension in Windows Explorer. The following table lists the most common types of files in a Flare project that you would scan:
