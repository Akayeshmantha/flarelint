Following and Breaking Rules

applies all its rules to each file in a Flare project. Either the file follows a rule, breaks a rule, or a rule is not applicable to the file.

adds a rule’s message to the report only when a file breaks the rule.

A file follows a rule when it meets all these requirements:

  • The file’s name has an extension specified in the rule’s extensions section.
  • The file has an element for which the when section is true. If the rule has more than one when section, FlareLint selects the element specified by the first when section that is true in the rule.
  • The test section gives true for the selected element. If a rule has more than one test section, FlareLint stops at the first true test section.

A file breaks a rule, and reports the rule’s message, when it meets all these requirements:

  • The file’s name has an extension specified in the rule’s extensions section.
  • The file has an element for which the when section is true. If the rule has more than one when section, FlareLint selects the element specified by the first when section that is true.
  • The test section gives false for the selected element. If the rule has more than one test section, all test sections must give false.

A rule is not applicable to a file when it meets at least one of these requirements:

  • The file’s name has an extension that is not specified in the rule’s extensions section.
  • There are no elements in the file for which the when section is true. If a rule has more than one when section, FlareLint stops when all when sections give false.
