Quoting Text

Some predicates require you to specify some text for an element’s content or an attribute’s value.

You can specify text by starting and ending it with straight quotation marks. For example the following predicate gives true for a p element ends with The End:

self-ends p "The End."

You must quote text on one line; does not accept quoted text that contains line endings. However, you can specify a line-ending with the characters “\n”.

For example, the following is incorrect:

"First line.
Second line."

Instead, you would type the following:

"First line.\nSecond line."

To quote a straight quotation mark (“"”)in, precede it with a backslash (“\"”).

For example, quote the following text:

"Ideally, I want to define the ideal." -- Plato

You would type the following in a rule file:

"\"Ideally, I want to define the ideal.\" -- Plato"

To specify a backslash (“\”), type it twice.

For example, to specify a Windows file path, you type backslash characters twice. To specify C:\Users\tinatwriter you would type the following:

"C:\\Users\\tinatwriter"
