Message Section

Specifies the text to insert in a report when an element fails a rule. A rule file must contain a single message section. The message section must be after the test sections.

When a matched element in a matched file fails all tests in a rule, considers the rule to be broken and inserts the rule's message in its report. In other words, this is the only part of a rule that a reader of a report might actually see.

Tip: A message should, at a minimum, answer the question "Why did the matched element break this rule?" Ideally, the message should also include a solution and other guidance.

For example:

message:
Missing period at the end of a sentence. To fix, insert a period.

ignores line breaks in a message. It places all text in a message into a single paragraph in the report.

For example, the following message section gives the same result in the report as the previous example:

message: Missing period at the end of a 
sentence. 
To fix, insert a period.

The message section can use simple markdown-like syntax. For bold, wrap text in asterisks (“*”). For monospace code, wrap text in back-ticks (“`”). For emphasis, wrap text in underscores (“_”).

If you want to use these special characters literally, you can prefix them with a backslash ("\").

For example, the following shows a message section and how it appears in a FlareLint report:

message:
Missing title (`h1`). *To fix:* add a title to the 
beginning of the topic body.
