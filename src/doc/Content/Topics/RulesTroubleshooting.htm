Troubleshooting Rules

Adding, removing, creating, and customizing rules can give you results that you do not expect.

Tip: You can view the name of the rule file from which an error was generated by hovering your mouse over the word "error" or "warning" in the message.

Use the following tips to help you diagnose and fix problems with rules.

Problem

Cause

Solution

ignores my rule in all my projects.

The file name for the rule does not end with rule.txt.

Rename the rule file then run again.

The rule file is not in your personal rule folder.

In Windows Explorer, move or copy your rule file to .

The rule’s extensions section does not specify the correct type of file name extenions. Or the extensions begin with periods (.).

Correct the extensions section then run . See Extensions Section.

ignores my rule in some projects.

The rule’s when expressions are false in unintended situations.

Or the rule’s test expressions are true in some unintended situations.

Compare the content of the projects that break your rule with the projects that do not break your rule. Correct your rule’s when or test expressions accordingly.

displays an error: Expected a x section, got "y" instead. A section must start at the beginning of a line.

The sections in the specified rule file are in an incorrect order or there is a missing section, x.

Edit the rule file so that sections are in the correct order and there are no missing sections. See Creating and Customizing Rules.

displays an error: Expected a predicate, got "x" instead.

An expression in a when or test section is incorrectly formed in the specified rule file. For example, there might be a missing predicate after and or not.

Edit the rule file so that the expression is properly formed. See Conditions and Predicates.

displays an error: the report contains badly formed HTML code.

When building the report as an HTML file, inserted or used incorrect HTML code. This incorrect HTML code might be from a mismatched markdown character in a rule message section or in a badly formed template for the HTML report.

If the message section of a rule file uses a “*”, “`”, or “_” character that you do not intend to start or end bold, monospace, or emphasized text, then insert a “\” character before it.

If you recently customized the TEMPLATE variables in the resources.py file, make sure your modifications result in correctly-formed HTML.
