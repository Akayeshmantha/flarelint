Creating and Customizing Rules

You can create new rules and modify existing ones. Each rule is defined in a rule file. A rule file is a plain text file that you can view and edit with any text editor, such as Microsoft Notepad. The name of a rule file ends with rule.txt. For example, the flarelint_h1_missing_rule.txt file specifies that in topic files, the body element must contain an h1 element.

A rule file is composed of sections, with each section starting with a section name. For example, the following is a comment section:

comment: See other rules for checking for the 
position of the header in the topic body.

The content of a section may have any amount of spaces and line breaks. You can use the extra spaces and line breaks to make it clearer for you to read. For example, the following specifies the same section as the previous example:

comment: 
  See other rules for checking for the 
  position of the header in the topic body.

There is only one exception to where to you can put extra spaces: a section name must start at the beginning of a line. Also, it must immediately end with a colon (“:”). Sometimes a section contains text that includes a colon but you do not intend it to start a section. In this case, make sure that word preceding the colon does not start at the beginning of a line.

comment: The following phrase does 
not start a new section: it is still 
in the comment section.

A rule file has the following sections. In a rule file, you must organize sections in the following order:

  • comment: A description, annotation, or other information for you and other people. ignores the text in the comment section. This section is optional.
  • rule: The type of rule, either error or warning. This section is required.
  • extensions: The file name extensions of the types of files to which to apply this rule. You must specify at least one file name extension. Do not include the dot (“.”) in the extensions. This section is required.
  • when: An expression that describes the HTML or XML element in a Flare file that this rule applies to. You must specify at least one when section. You can specify more when sections if necessary.
  • test: An expression that specifies requirements for the element that was matched in the when section. You must specify at least one test section. You can specify more test sections if necessary.
  • message: Text that describes why the test sections failed. This is the text that appears in a report. This section is required.
