Choosing Rules to Apply

You can specify which rules to apply and which rules to ignore by modifying the contents of your personal rule folder, .

loads a rule when it meets these requirements:

  • The file name of the rule ends with rule.txt.
  • The rule file is in your personal rule folder, .

ignores all other rule files.

Warning: Do not modify the contents of the rules folder in the package. Modifying the installed package could prevent you from upgrading and uninstalling .

To add rules:

  1. Use on any project. See Using.

    Running ensures that, at a minimum, the default rules get installed in your personal rule folder.

    You can ignore and delete the report that generates.

  2. In Windows Explorer, navigate to the folder that contains the rules to add to , then select them.

    For example, navigate to the shared folder that contains your team's standardized set of rules.

  3. Copy the rules into the clipboard.

  4. Click the Windows Start button and in the search field, type %APPDATA%\FlareLint then press Enter.

    A Windows Explorer window appears displaying the contents of your personal rule folder.

  5. Paste your rule files from your clipboard into your personal rule folder.

To specify which rules to ignore:

  1. Use on any project. See Using.

    Running ensures that, at a minimum, the default rules get installed in your personal rule folder.

    You can ignore and delete the report that generates.

  2. Click the Windows Start button and in the search field, type then press Enter.

    A Windows Explorer windows appears to display the contents of your personal rule folder.

  3. Select the files for the rules that you want to ignore.

  4. Do one of the following:

    • Move the selected files to a folder outside of your personal rules folder. For example, drag your selected files to your desktop.
    • To move the selected rules to the Recycle Bin, press the Delete key.
    • Rename the selected files to end with something other than rule.txt. For example, if you rename acme_rule.txt to acme_rule_IGNORE.txt, then ignores this rule.
